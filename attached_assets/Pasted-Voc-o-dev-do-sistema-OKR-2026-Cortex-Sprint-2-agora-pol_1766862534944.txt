REGRAS DE UI (PADRÃO DO SISTEMA)

- Nada de “vermelhão” gigante no dashboard: use fundo neutro + alertas com tint leve.
- Bordas, sombras e radius consistentes (ex: rounded-xl/2xl). Grid com espaçamento fixo.
- Para dados vazios (2026 ainda não rodou): sempre mostrar EMPTY STATE elegante (ícone + texto curto + sugestão do que fazer).
- Status padrão (para badges e colunas):
  - backlog/planned = cinza
  - doing/in_progress = azul
  - done = verde
  - blocked = vermelho
  - at_risk = amarelo (se existir)
- Badges devem aparecer SEMPRE em iniciativas e KRs.

- Seção “Alertas de Métricas”:
  - Ordenar por pior gap primeiro.
  - Cada alerta com: nome, meta, atual, % e badge de status.
  - Visual: fundo branco, borda leve, badge e ícone — sem bloco vermelho enorme.
- Adicionar “Foco do Trimestre”:
  - Um card/lista com as 6 principais iniciativas do trimestre (prioridade: blocked > doing > backlog).
  - Mostrar: título, objective badge, KR badges, owner pill, status badge.
  - CTA: “Ver todas iniciativas”.
- Adicionar “Resumo por Objetivo” (O1..O5):
  - Mini-cards com: % KRs no alvo, # iniciativas doing/blocked, e um indicador de risco.
- Melhorar o card “Evolução MRR”:
  - Se não tem dados: empty state.
  - Se tem dados: linha simples, sem poluição.

(2) INICIATIVAS — KANBAN + LISTA
- Manter Kanban com colunas: Backlog / Em andamento / Concluído / Bloqueado.
- Adicionar toggle: [Kanban] [Lista]
- VIEW LISTA (tabela):
  - Colunas: Iniciativa | Objetivo | Quarter | KRs | Owner | Status | Tags
  - Filtros no topo: Objetivo, Quarter, Owner, Status + Search
  - Ordenação: padrão por Status (blocked/doing primeiro) e depois por Objective/Quarter.
  - Row click abre Drawer/Modal de detalhes com:
    - descrição (se existir)
    - KRs linkados
    - checklist (subtarefas — simples)
    - histórico de updates (placeholder se ainda não existir)
- Kanban cards: melhorar visual:
  - Título forte
  - Linha de badges (Objective + Quarter + KRs)
  - Owner pill
  - Status badge claro
